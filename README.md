# 面向产品经理和 UX 设计师的高质量 AI Demo 页面生成 Skill

## 一、Skill 定位

本 Skill 用于辅助产品经理、UX 设计师在 vibe coding 场景下快速生成高质量 Demo 页面。通过预设统一的视觉规范、交互规范、组件规则、页面结构和业务表达方式，约束 AI 工具生成更接近真实产品设计稿和可交付 Demo 的页面效果，避免页面出现明显“AI 味”、模板化堆砌、视觉廉价、交互不合理、信息层级混乱等问题。

## 二、适用场景

当用户需要通过自然语言快速生成产品 Demo、业务页面原型、管理后台页面、数据看板、流程页面、配置页面、详情页面或功能验证页面时，应使用本 Skill。

典型使用场景包括：

1. 产品经理基于需求描述快速生成可演示页面；
2. UX 设计师基于业务场景快速生成页面初稿；
3. 项目团队在方案汇报、需求评审、客户演示中快速构建 Demo；
4. 在 Cursor、Lovable、v0、Bolt、Replit、Claude Code、Codex 等 vibe coding 工具中约束页面设计质量；
5. 将设计规范转化为可复用的前端页面生成规则。

## 三、核心目标

本 Skill 的目标不是简单生成页面，而是让 AI 生成的页面具备真实产品感、业务可信度和设计一致性。

具体目标包括：

1. 避免生成页面出现过度圆角、渐变滥用、玻璃拟态堆砌、大面积空洞卡片、无意义图标、AI 风格插画等常见“AI 味”；
2. 统一页面布局、颜色、字体、间距、组件、状态、交互反馈等基础设计规则；
3. 提升复杂业务页面的信息层级、可读性和操作效率；
4. 让生成结果更适合产品经理和 UX 设计师直接用于 Demo、评审和二次设计；
5. 将本次项目的设计经验沉淀为可复用的页面生成规范。

## 四、设计原则

### 1. 真实产品优先

页面应优先呈现真实业务系统的可信感，而不是展示炫技效果。设计应克制、稳定、清晰，避免为了视觉冲击而使用过度装饰。

### 2. 业务信息优先

页面设计应围绕用户任务展开，优先表达业务目标、关键数据、主要操作和状态变化，而不是堆砌无关模块。

### 3. 信息层级清晰

通过标题、分组、卡片、表格、标签、状态、操作区等方式建立明确的信息层级，使用户能够快速理解当前页面是什么、要看什么、能做什么。

### 4. 组件一致

按钮、表单、表格、弹窗、标签、筛选器、分页、状态提示、空状态、加载状态等组件应遵循统一规则，避免同一类操作在不同页面中表现不一致。

### 5. Demo 可演示

生成页面不仅要好看，还要能够支撑演示场景。页面中应包含合理的数据样例、操作路径、关键状态和结果反馈，方便用户讲清楚业务逻辑。
